# Common Game Sample Set (Genre-Agnostic)

Use this sample when genre-specific details are not available yet.

## Included Files

- `sample-report.md`: human-readable baseline architecture report
- `sample-report.json`: machine-readable companion output
- `sample-folder-tree.txt`: starter folder tree for game projects

## Purpose

- provide a stable baseline for Outgame(DDD) + Ingame(ECS)
- provide default multiplayer Port/Adapter boundary
- provide Unity UPM modularization baseline for Ads/Achievements/Social

## Usage

1. Start from `sample-report.md` sections.
2. Replace placeholder assumptions with project-specific facts.
3. Keep `sample-report.json` in sync when delivering dual-format output.
4. If migration is in scope, layer in a migration scaffold with subsystem classification, ambiguity register, and task board docs.
